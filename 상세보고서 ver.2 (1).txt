 요약문


 본 프로젝트는 AI 사주 분석 기술을 활용한 커플 애플리케이션 'Soulmatch'를 개발하는 것으로, 그 핵심 목표는 기념일 관리와 같은 단순 기능에 머무르는 기존 앱의 한계를 넘어 AI 궁합 분석 및 랭킹 시스템 등 참여형 콘텐츠를 통해 사용자에게 관계를 탐색하는 새로운 경험을 제공하는 것이다. 이를 위해 우리 조는 체계적인 평가를 거쳐 개발 기능의 우선순위를 결정했는데, 1차 가중순위결정법으로 시장성과 실현성 등을 고려해 서비스의 핵심 기능을 선정했고, 2차 퓨 매트릭스 분석으로 부가 가치가 높은 기능까지 최종 개발 범위에 포함시켰다. 이렇게 확정된 기능들은 구현 난이도에 따라 즉시 개발 과제와 장기 과제로 분류되며, 회원 관리, 궁합 분석, 랭킹 시스템 등은 우선적으로 개발하고 개인 성격 분석처럼 복잡도가 높은 기능은 향후 단계적으로 구현할 계획이다.


Abstract


 This project is dedicated to the development of 'Soulmatch,' a couple's application that utilizes AI-driven Saju analysis technology. Its core objective is to transcend the limitations of existing apps, which are often confined to simple functions like anniversary management. Instead, we aim to provide users with a novel experience to deeply explore their relationships through engaging content, including AI-powered compatibility analysis and a ranking system. To achieve this, our team has systematically prioritized development features through a structured evaluation. First, we used a weighted scoring method to select the service's core features, considering factors such as marketability and feasibility. Second, a Pugh matrix analysis was conducted to incorporate high value-added functions into the final development scope. These finalized features are categorized into immediate and long-term tasks based on their implementation difficulty. Core functionalities like user management, compatibility analysis, and the ranking system will be developed with priority. More complex features, such as individual personality analysis, are planned for phased implementation in the future. Ultimately, the final goal of this project is to provide profound value to users and to establish a sustainable, data-driven revenue model.

Ⅰ. 개요


A. 목적 및 주안점

 본 보고서는 'Soulmatch' 애플리케이션 개발 프로젝트의 상세 설계 단계 결과를 기술하는 것을 목적으로 한다. 「개념 설계 보고서」에서 도출된 핵심 기능과 개발 범위를 바탕으로, 실제 구현에 필요한 시스템 아키텍처, 데이터베이스 모델, 기능별 명세 등 기술적 사항을 구체적으로 정의한다.

 본 프로젝트의 핵심은 AI 사주 분석 기술을 활용하여 기존 커플 앱의 기능적 한계를 극복하는 데 있다. 사용자의 생년월일 정보로 궁합 지수를 산출하고, 이를 '랭킹 시스템' 및 '궁합 인증서'와 같은 참여형 콘텐츠와 연계함으로써 사용자가 관계를 깊이 있게 탐색하고 즐기는 새로운 경험을 제공하고자 한다. 이를 위해 우선순위가 높은 핵심 기능부터 개발하여 시장에 성공적으로 진입하고, 데이터 기반의 지속 가능한 서비스 모델을 구축하는 것을 목표로 한다.

B. 개발 아키텍처

 프로젝트의 개발 아키텍처는 프론트엔드와 백엔드가 명확히 분리된 구조를 채택한다. 이는 기제공된 Python 기반의 AI 사주 분석 코드를 백엔드의 핵심 로직으로 효과적으로 통합하기 위한 전략적 결정이다. 양측은 REST API를 통해 통신하며, 프론트엔드는 사용자 인터페이스(UI) 및 경험(UX) 구현에, 백엔드는 데이터 처리와 비즈니스 로직 수행에 집중하여 개발의 독립성과 효율성을 확보한다. 본 보고서 또한 이러한 구조에 따라 각 영역의 설계안을 나누어 제시한다.

C. 주요 기능 명세

 이번 프로젝트에서 구현할 주요 기능별 세부 설계 내용은 다음과 같다.

 가) 회원 관리 시스템

 회원 관리 시스템은 'Soulmatch' 서비스의 기본 토대를 이루는 기능으로, 사용자가 서비  스  를 이용하기 위한 계정을 생성, 로그인하고 파트너와 관계를 설정할 수 있도록 설계되었다. 회원가입은 사용자가 직접 설정하는 고유한 아이디와 비밀번호를 기반으로 진행되며, 닉네임 등 최소한의 정보만으로 계정을 생성할 수 있다. 자동 로그인 기능 역시 지원하여 사용자의 재방문 편의성을 높인다.

 프로필 관리 페이지에서는 닉네임 변경, 비밀번호 관리 등 기본적인 계정 설정이 가능하다. 특히, 이 시스템의 핵심은 상대방을 초대하고 연결하는 ‘파트너 연동’ 기능이다. 한 번 연동된 커플 정보는 별도로 관리되어 서비스 전반에서 활용된다. 백엔드에서는 사용자 정보를 Users 테이블에, 커플 연결 정보는 별도의 Couples 테이블에 저장하며, 비밀번호는 Bcrypt로 안전하게 해시하고 로그인 인증은 JWT(JSON Web Token)를 사용하여 보안을 강화한다. 이 과정에서 안정적인 인증 체계를 구축하고 사용자 데이터를 안전하게 보호하는 것이 중요하다.

 나) 핵심 궁합 분석 기능

  서비스의 핵심 가치를 담은 궁합 분석 기능은 프로젝트 시작 시에 제공된 Python 언어로 작성된 딥러닝 기반의 분석 엔진을 통해 구현된다. 이 분석 코드는 독자적인 백엔드 서버에 탑재되며, 애플리케이션(클라이언트)과는 REST API 형식으로 통신하여 데이터를 교환한다.

사용자가 커플의 생년월일시 정보를 입력하면, 앱은 이 데이터를 사전에 정의된 API 엔드포인트로 전송한다. 서버는 해당 요청을 받아 내부의 Python 분석 코드를 실행하여 궁합 점수 결과를 산출한다. 분석 결과는 표준 JSON 형식으로 구조화되어 클라이언트에 응답으로 반환되며, 동시에 AnalysisResults 테이블에 저장되어 사용자가 언제든 다시 조회할 수 있도록 구현된다.

이 기능 구현의 핵심 과제는 분석 로직의 연산 부하를 고려한 서버 성능 최적화와 안정적인 API 응답 시간을 보장하는 것이다. 또한, 다양한 입력값에 대한 예외 처리 로직을 견고하게 설계하고, 향후 사용자 증가에 대비하여 서버를 확장 가능하도록 구축하는 것이 중요하다.

 다) 분석 결과 시각화 및 공유 기능

 분석 결과 시각화 및 공유 기능은 AI가 도출한 다소 복잡할 수 있는 궁합 정보를 사용자가 쉽고 재미있게 소비하고, 이를 외부에 확산시켜 서비스의 바이럴 효과를 창출하기 위해 설계되었다. 단순히 숫자로만 결과를 보여주는 대신, 점수에 따라 채워지는 게이지 바 형태의 UI와 점수대에 따라 배경색이나 캐릭터의 표정이 변하는 동적인 시각 효과를 적용하여 감성적인 만족도를 높인다. 또한, 커플의 이름과 궁합 점수가 담긴 세련된 디자인의 '궁합 인증서' 이미지를 서버에서 동적으로 생성해주어, 사용자가 이를 이미지 파일로 저장하거나 카카오톡, 인스타그램 등 SNS로 직접 공유할 수 있도록 지원한다. 다양한 디바이스 환경에서도 이미지가 깨지지 않도록 반응형으로 설계하고, 이미지 생성으로 인한 서버 부하를 최소화하기 위한 캐싱 전략을 도입하는 것이 기술적인 고려 사항이다.


 라) 랭킹 및 참여형 콘텐츠

 랭킹 및 참여형 콘텐츠는 단순한 정보 조회를 넘어 사용자의 지속적인 참여와 경쟁을 유도하는 게임화(Gamification) 요소로서, 서비스 재방문율과 충성도를 높이는 핵심 동력이다. 사용자는 궁합 분석 후 '랭킹 등록하기' 버튼을 통해 자신들의 점수를 전체 랭킹 보드에 올릴 수 있고, 이때 다른 사용자에게 보여질 재치 있는 '한 줄 소개'를 함께 남길 수 있다. 랭킹 상위권에 오른 커플에게는 특별한 배지를 부여하고, 오프라인 데이트에 활용 가능한 제휴사 할인 쿠폰을 제공하여 실질적인 보상을 통한 참여 동기를 강화한다. 백엔드에서는 대량의 순위 데이터를 효율적으로 처리하기 위해 데이터베이스 인덱싱 최적화나 Redis의 Sorted Set과 같은 인메모리 데이터 스토어 활용을 검토한다. 공정한 경쟁 환경을 위해 단기간 내 중복 등록과 같은 어뷰징 행위를 방지하는 정책과 기술적 장치를 마련하는 것이 중요하다.

Ⅲ. Frontend


A. Frontend UseCase Diagram


 <그림 1 Frontend UseCase Diagram>

 본격적인 Frontend 시스템 설계에 앞서, UI에 대한 사용자 요구사항을 명확히 하고 기능적 범위를 정의하고자 Use Case 다이어그램을 작성했다. 사용자는 크게 '회원'과 '비회원'으로 구분되며, 각기 다른 접근 권한과 사용 시나리오를 가진다.

먼저, 비회원은 데이터베이스에 사용자별 기록을 생성하거나 수정할 필요가 없는 기능에 접근할 수 있다. 이에 따라 '궁합 결과 조회'나 '랭킹 조회' 같은 단순 조회 기능은 가입 없이도 자유롭게 이용 가능하다. 또한, 조회된 결과를 '이미지로 저장'하거나 'SNS로 공유'하는 파생 기능(<<extend>>) 역시 비회원에게 제공된다. 이러한 개방적인 기능 제공은 비회원이 서비스의 핵심 가치를 충분히 체험하게 하여, 자연스럽게 회원가입으로 이어지도록 유도하는 역할을 한다.

반면, 회원은 '로그인'을 통해 식별된 사용자로서 모든 기능에 대한 접근 권한을 가진다. 특히 '랭킹 등록'이나 '이벤트 참여'와 같이 개인화된 데이터를 데이터베이스에 기록하고 관리해야 하는 기능들은 사용자 식별이 필수적이다. 따라서 이러한 기능들은 '로그인'을 필수 전제조건(<<include>>)으로 하는 회원 전용으로 설계되었다. 이 설계는 기술적 요구사항을 충족하는 동시에, 회원에게만 제공되는 참여형 콘텐츠를 통해 서비스에 대한 몰입도와 충성도를 높이는 부가적인 효과를 기대할 수 있다. 이 외에도 '궁합 결과 조회' 시 '궁합 분석 요청'이, '프로필 관리' 시 '파트너 연동'이 포함되는 등 기능 간의 논리적 선후 관계 또한 명확히 정의하였다.


B. 구현 언어 및 기술 스택

 프론트엔드의 경우 모바일 이용자의 접근성을 고려하여 애플리케이션 형태로 하되, iOS와 Android 모두의 접근성과 개발 편의성을 위해 React와 유사한 하이브리드 앱 프레임워크인 React Native를 이용하기로 한다. 
















C. Frontend Class Diagram


 
<그림 2 Frontend Class Diagram>

본격적인 Frontend 시스템 설계에 앞서, UI에 대한 사용자 요구사항을 명확히 하고 기능적 범위를 정의하고자 Use Case 다이어그램을 작성했다. 사용자는 크게 '회원'과 '비회원'으로 구분되며, 각기 다른 접근 권한과 사용 시나리오를 가진다.
비회원은 데이터베이스에 사용자별 기록을 생성하거나 수정할 필요가 없는 기능에 접근할 수 있다. 이에 따라 '궁합 결과 조회'나 '랭킹 조회' 같은 단순 조회 기능은 가입 없이도 자유롭게 이용 가능하다. 또한, 조회된 결과를 '이미지로 저장'하거나 'SNS로 공유'하는 파생 기능(<<extend>>) 역시 비회원에게 제공된다. 이러한 개방적인 기능 제공은 비회원이 서비스의 핵심 가치를 체험하게 하여, 자연스럽게 회원가입으로 이어지도록 유도하는 역할을 한다.
반면, '회원'은 '로그인'을 통해 식별된 사용자로서 모든 기능에 대한 접근 권한을 가진다. 특히 '랭킹 등록'이나 '이벤트 참여'와 같이 개인화된 데이터를 데이터베이스에 기록하고 관리해야 하는 기능들은 사용자 식별이 필수적이다. 따라서 이러한 기능들은 '로그인'을 필수 전제조건(<<include>>)으로 하는 회원 전용으로 설계되었다. 이 설계는 기술적 요구사항을 충족하는 동시에, 회원에게만 제공되는 참여형 콘텐츠를 통해 부가적인 효과를 기대할 수 있다.


Ⅳ. Backend

A. Backend UseCase Diagram

<그림 3 Backend UseCase Diagram>


 본격적인 Backend 시스템 설계에 앞서, UI에 대한 사용자 요구사항을 명확히 하고 기능적 범위를 정의하고자 Use Case 다이어그램을 작성했다.

 본 유스케이스 다이어그램은 ‘Soulmatch 애플리케이션’의 주요 기능 흐름과 사용자 역할 간 상호작용을 시각화한 것이다. 시스템의 주요 행위자는 사용자(User), 관리자(Admin), 그리고 외부 SNS 서비스(SNS)이며, 모든 기능은 중앙의 백엔드 시스템(Backend) 내에서 처리된다. 사용자는 회원가입 및 로그인 과정을 통해 인증된 후, 프로필 관리, 파트너 연동/해제, 궁합 점수 계산, 분석 결과 조회, 인증서 이미지 생성 등의 주요 기능을 수행할 수 있다. 또한 SNS 공유 기능을 통해 외부 플랫폼으로 분석 결과를 전송할 수 있으며, 자동 로그인과 토큰 재발급 기능은 사용자 편의성을 높이기 위해 로그인 기능과 연동되어 있다.
 관리자는 시스템의 정확도 검증(성능·정확도 점검)과 보상/쿠폰 관리를 담당하며, 이를 통해 사용자 이벤트 참여 현황과 랭킹 보드 데이터를 유지·관리한다. 각 기능은 독립적인 유스케이스로 구분되지만, JWT 기반의 인증 및 Bcrypt 비밀번호 해시 등 보안 요소가 모든 기능의 기반으로 포함되어 있다. 외부 SNS와의 API 연동은 서비스 확장성과 사용자 참여 유도를 위한 핵심 부분이며, 전체적으로 이 다이어그램은 사용자 중심의 서비스 흐름과 백엔드 구조 간의 유기적 관계를 명확히 표현하고 있다.

 이해를 돕기 위해 Backend Architecture Diagram으로 전체적인 백엔드 시스템의 구조를 표현하였다,



C. Backend Class Diagram

 백엔드 클래스 다이어그램은 시스템의 복잡한 구조를 명확히 표현하기 위해 도메인 모델과 서비스·리포지토리 계층으로 구분하였다. 도메인 모델은 서비스의 핵심 데이터를 정의하고 객체 간 관계를 구조화하여 데이터 일관성을 유지하기 위한 기반이 되며, 서비스·리포지토리 계층은 실제 비즈니스 로직과 데이터 접근을 담당하여 기능 단위의 처리 흐름을 명확히 한다. 이러한 구분을 통해 데이터 구조와 로직을 분리함으로써 유지보수성과 확장성을 높이고, 역할에 따라 책임이 명확한 계층적 설계를 구현하고자 하였다.



<그림 5 Backend Class Diagram – Domain model>




 본 도메인 모델은 사용자를 중심으로 커플 관계, 궁합 분석, 랭킹, 보상 및 이벤트, 공유 로그 등 주요 기능을 구조화한 백엔드 핵심 데이터 모델이다. User는 기본 회원 정보를, Profile은 생일·별자리 등 추가 정보를 관리하며 1:1로 연결된다. Couple은 두 사용자의 관계를 나타내고, 이를 기반으로 AnalysisRequest와 AnalysisResult가 궁합 점수를 계산·저장한다. RankingEntry는 커플별 점수를 주기별(DAILY, WEEKLY 등)로 집계하고 Redis TTL을 통해 캐시 관리가 가능하도록 설계되었다.
 보상 시스템은 Badge, Coupon, Event 클래스를 중심으로 구성되어 있으며, 사용자의 활동에 따라 뱃지나 쿠폰 형태의 리워드를 부여한다. UserBadge는 뱃지 획득 이력을, Coupon은 소유 및 사용 여부를 관리한다. 또한 ShareLog를 통해 사용자의 SNS 공유 내역을 기록하여 서비스 참여도와 확산 효과를 추적할 수 있다. 전체적으로 본 모델은 주요 기능 간의 관계를 명확히 정의해 데이터 일관성과 유지보수성을 높이는 것을 목표로 한다.




















<그림 6 Backend Class Diagram – Service-Repository Structure>

 본 다이어그램은 백엔드의 서비스 계층(Service Layer) 과 저장소 계층(Repository Layer) 간의 구조적 의존 관계를 나타낸다. 각 서비스는 비즈니스 로직을 담당하며, 관련 리포지토리와 연동해 데이터를 처리한다. AuthService는 사용자 인증과 토큰 관리를, UserService는 회원 및 커플 데이터를 관리한다. AnalysisService는 궁합 분석 요청을 처리하며 PythonEngineClient와 MessageQueue를 통해 외부 연산을 수행하고 결과를 저장한다.
 ImageService는 분석 결과를 이미지로 변환하고 캐시를 활용해 성능을 향상시키며, RankingService는 캐시와 연계해 순위 정보를 효율적으로 관리한다. EventService는 이벤트·쿠폰 데이터를, ShareService는 SNS 공유 및 로그를 관리한다. 이러한 구조는 각 기능 간 결합도를 낮추고, 외부 모듈과의 의존성을 분리하여 시스템의 유지보수성과 확장성을 높이도록 설계되었다.
